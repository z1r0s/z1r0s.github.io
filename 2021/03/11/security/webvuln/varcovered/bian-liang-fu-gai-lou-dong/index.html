变量覆盖

​ 变量覆盖漏洞是指自定义的参数值替换原有变量值的情况。导致变量覆盖主要是由于开启了全局变量注册功能或$$、extract()、parse_str()、import_request_variables()使用不当造成的。通常来说，单独的变量覆盖漏洞很难有利用价值，但是在与其他应用代码或漏洞结合后，其造成的危害可能是无法估量的。例如：购买商品的支付系统若存在变量覆盖漏洞，就可能存在0元支付下单的情况；文件上传页面，覆盖掉白名单的列表，导致任意文件上传等。

产生原因

  1. 开启了全局变量注册功能
    • register_globals=On
  2. 可变变量使用不当
    • $$
  3. 函数使用不当
    • extract()
    • parse_str()
    • import_request_variables()

漏洞利用

register_globals

​ Register_globals设置为On时，默认会自动将GET、POST、COOKIE中的参数注册为全局变量。Register_globals选项在5.2默认为On；Register_globals选项在5.3默认为Off；Register_globals选项在5.4之后被废弃。

测试代码：

echo "Register_globals: " . (int)ini_get("register_globals") . "<br/>";
// echo '$_GET["id"] :' . $_GET['id'] . "<br/>";
// echo '$_POST["id"] :' . $_POST['id'] . "<br/>";
echo '$_COOKIE["id"] :' . $_COOKIE['id'] . "<br/>";
echo '$id :' . $id;

可变变量

​ $$是一种可变变量的写法，它可以使一个普通变量的值作为可变变量的名字，这种类型常常会使用遍历的方式来释放变量的代码，最常见的就是foreach的遍历。如下代码所示是一个最常见的生成可变变量的方式。

测试代码1：

//  $$注册变量
$arr = array('_GET', '_POST', '_COOKIE');
// 注册变量
foreach ($arr as $item) {
    foreach ($$item as $k => $v) { // foreach($_GET as $k=>$v)
        $$k = $v; // $id = '1';
    }
}
$id = isset($id) ? $id : 2;
if ($id == 1) {
    echo "flag{xxxxxx}";
    die();
}
echo $id;

运行结果：

上述代码中只展现了$$声明可变变量的能力，但是没有体现出变量覆盖漏洞，下面将展示另一段代码具体体现$$可能引起的变量覆盖漏洞。

<?php
//  $$导致的变量覆盖
include 'flag.php';
$yds = "dog";
$is = "cat";
$handsome = 'yds';

foreach ($_GET as $x => $y) { //get传值
    $$x = $$y;  //漏洞在这里  比如输入 yds=flag 相当于 $yds=$flag
}
foreach ($_GET as $x => $y) {
    if ($_GET['flag'] === $x && $x !== 'flag') { //判断get传进来的值等不等于flag 如果等于flag则跳过
        exit($handsome);
        // handsome=flag&flag=handsome
    }
}
if (!isset($_GET['flag']) && !isset($_POST['flag'])) {
    exit($yds);
    // ?yds=flag
}
if ($_POST['flag'] === 'flag'  || $_GET['flag'] === 'flag') {
    exit($is);
    // ?is=flag&flag=flag
}

echo "the flag is: " . $flag;

extract()

函数说明

// 从数组中将变量导入到当前的符号表必须使用关联数组
extract ( array &$array , int $flags = EXTR_OVERWRITE , string $prefix = "" ) : int
- &$array: 必须使用关联数组
- $flags: 对待非法／数字和冲突的键名的方法将根据取出标记 flags 参数决定
    - EXTR_OVERWRITE：存在冲突覆盖（默认）
    - EXTR_SKIP：存在冲突不覆盖
    - EXTR_PREFIX_SAME：存在冲突新变量添加前缀
    - EXTR_PREFIX_ALL：数组中所有的变量都添加前缀
    - EXTR_IF_EXISTS：存在冲突覆盖变量，不存在冲突则不定义新变量
- $prefix: 存在冲突时若指定前缀，必须设置此参数

代码示例：

$a = "Original";
$my_array = array("a" => "Cat", "b" => "Dog", "c" => "Horse");
extract($my_array);
// 存在冲突覆盖（默认）
extract($my_array, EXTR_OVERWRITE);
// 存在冲突不覆盖
extract($my_array, EXTR_SKIP);
// 存在冲突新变量添加'test_'前缀
extract($my_array, EXTR_PREFIX_SAME, 'test');
// 数组中所有的变量都添加前缀
extract($my_array, EXTR_PREFIX_ALL, 'test');
// 存在冲突覆盖变量，不存在冲突则不定义新变量
extract($my_array, EXTR_IF_EXISTS);
// var_dump($GLOBALS);
// die();
//$a == Original || $a == Cat
echo "\$a = $a; \$b = $b; \$c = $c";

测试源码1：

<?php
// 变量覆盖漏洞

$flag = 'flag.php';

// echo $ceshi;
if (isset($ceshi)) {
    $content = trim(file_get_contents($flag));
    if ($ceshi == $content) {
        echo 'flag{xxxxxx}';
    } else {

原理分析

http://10.10.10.138/test.php?ceshi=&flag=

http://10.10.10.138/test.php?ceshi=1&flag=http://10.10.10.138/1.txt
- file_get_contents()函数可读URL读取一个可控的文件能满足判断条件

测试源码2：

<?php
// 过滤键中含有flag的所有参数
function test_waf($a)
{
    foreach ($a as $key => $value) {
        if (preg_match('/flag/i', $key)) {
            exit('are you a hacker');
        }
    }
}

foreach (array('_POST', '_GET', '_COOKIE') as $__R) {
    // 判断全局变量是否非空
    if ($$__R) {
        // 遍历所有参数
        foreach ($$__R as $__k => $__v) {
            // 变量已经被定义且键值相等，则删除该变量
            if (isset($$__k) && $$__k == $__v) {
                unset($$__k);
            }
        }
    }
}
// 需要绕过
if ($_POST) {
    test_waf($_POST);
}
if ($_GET) {
    test_waf($_GET);
}
if ($_COOKIE) {
    test_waf($_COOKIE);
}

// 得到$_GET
if ($_POST) {
    // 注册全局变量，冲突跳过
    extract($_POST, EXTR_SKIP);
}
if ($_GET) {
    // 注册全局变量，冲突跳过
    extract($_GET, EXTR_SKIP);
}
// 突破点
if (isset($_GET['flag'])) {
    if ($_GET['flag'] === $_GET['daiker']) {
        exit('error');
    }
    if (md5($_GET['flag']) == md5($_GET['daiker'])) {
        include($_GET['file']);
    }
}

parse_str()

函数说明

// 将字符串解析成多个变量
parse_str ( string $string , array &$result ) : void
- $string: 输入的字符串（参数以&分割）
- &$result: 若设置该变量，变量将会以数组元素的形式存入到这个数组

代码示例

$str = 'a=1&b=2&c=3';
parse_str($str);
echo '$a -> ' . $a . ' $b-> ' . $b . ' $c -> ' . $c;
// 将结果放到$res数组中
parse_str($str, $res);
print_r($res);

// 输出
$a -> 1 $b-> 2 $c -> 3
Array ( [a] => 1 [b] => 2 [c] => 3 )

测试源码：

<?php
error_reporting(0);
$a = "www.bihuo.cn";
$id = $_GET['id'];
@parse_str($id); //定义全局变量 7.2废弃无参遍历
var_dump($a);
if ($a[0] != "QNKCDZO" && md5($a[0]) == md5("QNKCDZO")) {
    echo "success";
} else {
    exit("failure");
}

使用如下所示payload，将输出success

http://10.10.10.138/test.php?id=a[]=240610708

// 输出
seccess

原理分析

http://10.10.10.138/test.php?id=a[]=240610708

- php属于弱类型语言，通过parse_str()函数覆盖变量a并将其该为数组类型并设置首个值为240610708
- md5("240610708") == md5("QNKCDZO")

import_request_variables()

函数说明:

// 将 GET/POST/Cookie 变量导入到全局作用域中
import_request_variables ( string types [, string prefix]) : bool
- types: 参数指定需要导入的变量（'G'、'P'和'C'分别表示 GET、POST 和 Cookie）//大小写不敏感
- prefix: 参数作为变量名的前缀，置于所有被导入到全局作用域的变量之前

代码示例：

<?php
// 变量覆盖漏洞

// import_request_variables() //4.1引入 5.4移除
// 使用之前先将register_globals参数设置为Off
import_request_variables("gpc");
echo $a;
// 为所有新创建的全局变量加上前缀
import_request_variables("gpc", 'test_');
echo $test_a;

运行结果：

漏洞防范

  • 在php.ini文件中设置register_globals=Off；
  • 使用原始变量数组，如$_POST,$_GET等数组变量进行操作；
  • 不使用foreach语句来遍历$_GET变量，而改用[(index)]来指定；
  • 验证变量是否存在，注册变量前先判断变量是否存在。
