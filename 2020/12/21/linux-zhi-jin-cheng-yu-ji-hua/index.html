Linux之进程与计划

Linux
操作系统
发布日期:   2020-12-21

进程管理

相关概念

  • 进程：动态的，被加载到内存里。
  • 程序：静态的，存在硬盘。
  • PID：进程唯一标识，通常结束进程时使用
  • 父子进程：每个进程可以创建一个或多个进程

进程查看

pstree

# 以树形方式显示进程之间的父子关系
pstree
参数：
 -a 显示完整信息
 -u 显示用户名
 -p 显示PID号

# 显示更加详细，会吧每个进程都显示出来。
pstree  -p

[root@CentOS7 ~]# pstree -aup | grep httpd
|-httpd,3102 -DFOREGROUND
|   |-httpd,3103,apache -DFOREGROUND
|   |-httpd,3104,apache -DFOREGROUND
|   |-httpd,3105,apache -DFOREGROUND
|   |-httpd,3106,apache -DFOREGROUND
|   `-httpd,3107,apache -DFOREGROUND
|           |-grep,3209 --color=auto httpd

pgrep

pgrep
参数：
 -o:仅显示找到的最小(起始)进程号;
 -n:仅显示找到的最大(结束)进程号;
 -l:显示进程名称;
 -P:指定父进程号;
 -g:指定进程组;
 -t:指定开启进程的终端;
 -u:指定进程的有效用户ID。
命令：
# 找到服务名称为httpd的服务,并且运行用户是apache的进程ID
psgrep  -l  httpd  -u  apache

ps

# 查看所有进程信息
ps -aux
结果描述：
 USER(用户)        
 PID(进程号) 
 %CPU %MEM：CPU和内存的使用率
 TTY ：连接终端（tty表示本地连接；pts表示远程登录）
 STAT ：当前进程的状态；S（sleep）,s(有很多子进程)，z（zombie僵尸进程）
 START：进程开始的时间
 COMMAND：进程正在执行的命令

# 查看父子进程之间的关系
ps -elf
说明：
 其中PPID代表父进程。如果为0，表示没有父进程。

top

# 动态显示进程的变化。退出q。
top

# 可适当介绍top工具的命令按键：P、M、N、h、q
 P:根据CPU占用情况对进出列表进行排序
 M:根据内存占用情况进行排序
 N:根据启动时间进行排序
 h:查看帮助信息
 q:退出top程序
 t:切换进程和CPU状态信息
 i:忽略显示和僵尸进程,这时一个开关式命令
 k:终止一个进程
 空格:强制更新进程状态并显示

# 命令举例
    top - 09:44:56 up 16 days, 21:23,  1 user,  load average: 9.59, 4.75, 1.92
    Tasks: 145 total,   2 running, 143 sleeping,   0 stopped,   0 zombie
    Cpu(s): 99.8%us,  0.1%sy,  0.0%ni,  0.2%id,  0.0%wa,  0.0%hi,  0.0%si,  0.0%st
    Mem:   4147888k total,  2493092k used,  1654796k free,   158188k buffers
    Swap:  5144568k total,       56k used,  5144512k free,  2013180k cached

    #第一行:系统当前时间;已经运行了16天21小时,1个用户登录;系统5/10/15分钟之内的平均负载
    #第二行:Tasks(总进程数是145);2个正运行的进程;143个睡眠的进程;0个停止;0个僵尸进程
    #第三行:CPU(用户控件占用CPU的百分比是99.8%),内核占用CPU0.1%;0.0%ni[用户进程空间内改变过优先级的进程占用CPU百分比];0.2%id[空闲CPU百分比], 0.0%wa[等待输入输出的CPU时间百分比];
    #第四行:Mem(内存)总大小是4G;2.4G被使用;1.6G空闲;158M作为内核缓存(buffer:写入缓冲区);
    #第五行:Swap(交换分区)总大小5G;用了56k;其余闲置;2G作为读取缓存(cached:CPU刚使用的数据);

前台切换

# 前后台切换命令
&:        加在一个命令最后,把进程放到后台执行
ctrl+z:    挂起任务到后台,暂停执行.比如某进程执行需要很长时间有比较消耗资源;可以暂停.
jobs:    查看后台运行的任务列表
fg:        将后台中的命令调到前台执行;fg  num(jobs查看的任务编号)
bg:        将暂停在后台的命令继续在后台执行

终止进程

# 终止进程：

#终止前台进程
Ctrl+C

# #终止后台进程;-u指定用户
kill 进程号
# 强制杀死
kil -9 进程号   

# 按照进程名杀死进程
pkill 名字
# 使用进程名杀死一组进程;-l不区分大小写;-u指定杀死用户的进程
killall  httpd

查看端口

# 显示所有端口号
netstat  -antpu
参数：
-a  显示所有  
-n  以数字的方式显示
-p  显示pid
-tu 表示TCP和UDP协议（“语言”）

# 查看22端口的连接（访问）情况
lsof -i:22

计划任务

一次性任务

  • 命令是at，支持的服务是atd
#1.#确认atd服务是开启的(默认开启)
service atd status
/etc/init.d/atd status
systemctl status atd

#2.一次性任务举例：
at 08:55
at> echo "123456" >/test/123.txt
at> echo aaaaaa >>/test/123.txt
at> echo bbbb >> /test/123.txt
at> （Ctrl+D退出编辑；Ctrl+C是终止编辑）

at 10:00 2019-12-14   #明天要执行的任务
>命令1
>命令2 
>(Ctrl+D退出编辑)

#3.检查任务
tail  -F   /test/123.txt  #-F可以动态追踪一个文件的变化

#4.查看和删除任务
atq   #查询未执行的任务
at -c 12 | tail   #-c查看任务需要执行的脚本 
atrm 12   #删除任务编号

系统任务分类

系统任务分为2大类：系统任务调度、用户任务调度。

  • 系统任务调度

    # 系统任务调度:
# 系统周期性所要执行的工作,比如写缓存数据到硬盘、日志清理等。
# 在/etc目录下有一个crontab文件,这个就是系统任务调度的配置文件。
cat  /etc/crontab 

##定义crond的环境变量
SHELL=/bin/bash    #指定shell解析器
PATH=/sbin:/bin:/usr/sbin:/usr/bin  #指定命令执行路径
MAILTO=root    #mailto发送邮件给root用户;如果mailto为空就不发送任务执行信息给用户;
HOME=/         #第四行HOME指定了执行命令的主目录

crontab执行轮询的脚本目录

[root@CentOS7 ~]# ll -d /etc/cron*ly
drwxr-xr-x. 2 root root 57 Oct 9 2018 /etc/cron.daily
drwxr-xr-x. 2 root root 22 Jun 10 2014 /etc/cron.hourly
drwxr-xr-x. 2 root root 6 Jun 10 2014 /etc/cron.monthly
drwxr-xr-x. 2 root root 6 Jun 10 2014 /etc/cron.weekly


- 用户任务调度

  ```shell
  # 用户任务调度文件
  # 所有用户定义的crontab都会被保存在/var/spool/cron目录中,文件名会以用户名命名.
  [root@CentOS7 ~]# ll /var/spool/cron/

  [root@CentOS7 ~]# ll /etc/cron.deny    #禁止用户使用crontab任务

周期性任务

周期性任务：crond用来周期性执行任务的服务。

#1.crond服务管理
在Centos6之前：
/etc/init.d/crond  status  
或者
service crond status
在centos7中：
systemctl status/enable/start crond.service  //其中enable是设置开机自启动


#2.crond的配置文件
#某个用户的任务计划
/var/spool/cron/用户名
#任务的日志文件(可以用来跟踪任务执行结果)
/var/log/cron


#3.crond常用命令
crontab  -e   #编辑任务
crontab  -l   #列出任务
crontab  -r   #删除任务 
crontab  -u   #指定任务用户

#4.crontab命令的基本格式
cat  /etc/crontab
# For details see man 4 crontabs

# Example of job definition:
# .---------------- minute (0 - 59)
# |  .------------- hour (0 - 23)
# |  |  .---------- day of month (1 - 31)
# |  |  |  .------- month (1 - 12) OR jan,feb,mar,apr ...
# |  |  |  |  .---- day of week (0 - 6) (Sunday=0 or 7) OR sun,mon,tue,wed,thu,fri,sat
# |  |  |  |  |
# *  *  *  *  * user-name  command to be executed

#crontab中特殊时间的表示方法
*   该范围内的任意时间
-   连续的时间范围
,   间隔的不连续时间
/   时间间隔的频率
举例:
0  17  *  *  1-5        周一到周五每天17:00 
30  8  *  *  1,3,5        每周一、三、五的8点30分
0  8-18/2  *  *  *        8点到18点之间每2小时
0  *  */3  *  *            每3天

crontab任务举例：

#1.每分钟打印一次自己自己名字到
*/1 * * * *    /bin/echo "Tom" >> /test/log/log.txt   2>&1
*   * * * *    /bin/ echo "Tom" >> /test/log/log.txt  2>&1

#2.上午07-11点每个小时运行一次
*   07-11  *  *  *    /etc/init.d/httpd  reload  #上午07-11点每分钟运行一次
00  07-11  *  *  *    /etc/init.d/httpd  reload  

#3.每天的7点/11点/13点各运行一次
00   07，11，13  *  *  *     
00   23,00-07/1  *  *  *    #不规范，每天的23点，00-07之间每个小时重启一次httpd；1写不写都行

#4.每天晚上23点和00-07之间每小时重启一次httpd服务
*   23,00-07/1  *  *  *     #每天的23点，00-07之间每小时的每分钟重启一次httpd
00   23,00-07  *  *  *      #正确：每天的23点，00-07之间每个小时重启一次httpd

#5.每天晚上12点重启一次httpd
00   00   *  *  *   /etc/init.d/httpd  reload    #每天的晚上12点重启一次httpd

crontab编写的步骤

#1.第一步:先在命令行测试命令是否可用

11 Feb 18:03:26 ntpdate[4752]: step time server 120.25.115.20 offset 1.005005 sec

#2.第二步:编写任务
[root@CentOS7 ~]# crontab   -e
*/5  * * * *   /usr/sbin/ntpdate ntp1.aliyun.com
