安装DNS服务器角色

环境描述：

​ “sbwyl”公司新组建了一个内部局域网，需要一台DNS服务器为内部用户提供域名解析服务，该如何搭建DNS服务器

具体步骤如下：

  • 首先从虚拟机开启一台Server 2008 R2的服务器用于搭建DNS服务器；
  • 在开启一台Windows 7的计算机用于模拟客户端；
  • 分别给服务器和客户机配置IP地址，在客户机配置IP地址时DNS要指向服务器IP地址；
  • 关闭两个系统的防火墙，并且能够互相拼通。

  1. 打开【服务器管理器】管理界面 → 角色 → 添加角色 → 在弹出“开始之前对话框”中点击【下一步】 → 在“选择服务器角色”对话框中选择【DNS服务器】单击左边复选框，操作结果如下图所示。

  2. 点击【下一步】按钮，显示DNS服务器简介界面，显示结果如下图所示。

  3. 点击【下一步】按钮，进入安装确认界面，显示结果如下图所示。

  4. 点击【下一步】按钮，显示DNS安装进度界面，显示结果如下图所示。

  5. 显示如下图所示表示DNS服务安装成功。

创建正向查询域

  1. 打开DNS管理器。开始 → 管理工具 → 选择【DNS】，DNS管理器界面如下图所示。

  2. 右键【WIN-H69V7LS 7Q7D】选择【新建区域…】选项，显示新建区域向导如下图所示。

  3. 点击【下一步】按钮进入区域类型选择界面，这里选择【主要区域】选项，显示结果如下图所示。

  4. 点击【下一步】按钮进入正向或反向查询区域设置，这里选择【正向查找区域】选项，操作结果如下图所示。

  5. 点击【下一步】按钮后进入区域名称设置界面，设置区域名称为sbwyl.com操作结果如下图所示。

  6. 点击【下一步】按钮进入区域文件设置界面，这里选择创建新文件选项操作结果如下图所示。

  7. 点击【下一步】按钮，进入动态更新设置界面，这里采用默认设置，显示结果如下图所示。

  8. 点击【下一步】按钮进入设置信息核对界面，显示结果如下图所示。

  9. 点击【完成】按钮后，操作结果显示如下图所示。

创建反向查询域

  1. 右键【反向查询区域】选择【新建区域…】选项，显示新建区域向导如下图所示。

  2. 点击【下一步】按钮进入区域类型选择界面，这里选择【主要区域】选项，显示结果如下图所示。

  3. 点击【下一步】按钮进入反向查询区域设置，这里选择【IPv4反向查找区域】选项，操作结果如下图所示。

  4. 点击【下一步】按钮，在新页面中设置网络ID为192.168.1，操作结果如下图所示。

  5. 点击【下一步】按钮，进入区域文件设置界面，采用默认设置，操作结果如下图所示。

  6. 点击【下一步】按钮，进入动态更新设置界面，这里采用默认设置，显示结果如下图所示。

  7. 点击【下一步】按钮进入设置信息核对界面，显示结果如下图所示。

  8. 点击【完成】按钮后，操作结果显示如下图所示。

创建主机记录

​ 假设sbwyl公司有一台计算机的FQDN（“FQDN”的意思是：正式域名）为：www.sbwyl.com，IP地址是192.168.1.222，如何在区域“sbwyl.com”下创建该主机记录

  1. 新建主机。在DNS管理器的左边窗口中右键【正向查询区域】下面的【sbwyl.com】，选择【新建主机…】选项操作界面如下图所示。

  2. 新建别名。在DNS管理器的左侧窗口中右键【sbwyl.com】选择【新建别名…】选项，操作结果如下图所示。

  3. 新建指针。右键【1.168.192.in-addr.arpa】选择【新建指针…】选项，操作结果如下图所示。

新建辅助区域

​ “sbwyl”公司规模做大为了减轻单台DNS的负载，现在需要将本台DNS服务器的内容保存到另一台DNS服务器中。

  1. 在Server2中配置IP地址，DNS地址要指向Server1服务器的IP地址。

  2. 在Server2中上安装“DNS”服务器后右键【正向查找区域】后选择【新建区域】

  3. 右键【下一步】按钮，进入区域设置界面，选择【辅助区域】，操作结果如下图所示。

  4. 点击【下一步】按钮，显示区域名称设置界面，操作结果如下图所示。（注：此处输入的区域名称要和源区域完全相同）

  5. 点击【下一步】按钮，进入主DNS设置界面，操作结果如下图所示。

  6. 点击【下一步】按钮，显示区域设置核对界面，显示结果如下图所示。

  7. Server1区域传送设置。在Server1中打开“DNS管理器”，右键sbwyl.com选择【属性】选项，点击【区域传送】选项卡，设置服务器IP地址操作结果如下图所示。

新建子域

​ “sbwyl”公司在北京成立了分公司，公司网络由总公司直接管理，管理员想通过部门进行管理DNS记录。

  1. 打开DNS服务器右键“sbwyl.com”选择【新建域…】选项，在显示的界面中设置子域名称，操作结果如下图所示。

  2. 在新建的域中可以新建主机和别名。

新建委派

​ 公司有一台DNS服务器，其内部局域网使用“sbwyl.com”作为域名，现在该公司在上海成立分公司，上海公司使用专线和总公司连接，上海分公司计划使用“sh.sbwyl.com”作为域名别切在本地解析。

  1. 右键“sbwyl.com”选择【新建委派…】选项，显示新建委派向导如下图所示。

  2. 点击【下一步】按钮，在受委派域名设置界面中设置委派的域，操作结果如下图所示。

  3. 点击【下一步】按钮，进入名称服务器设置界面，设置名称服务器操作结果如下图所示。（Server2服务器主机地址，用来模拟分公司）

  4. 点击【下一步】按钮，显示新建委派完成界面，显示结果如下图所示。

  5. 新建委派结果如下图所示。

转发器

​ 公司有一台DNS服务器和一个外部DNS服务器，为了安全内网DNS服务器不能和外网通信，但是配置内网DNS服务器IP地址的员工经常需要访问外网网站。

  1. 打开“DNS管理器”，右键：“CC”，选择【属性】选项，打开【转发器】转发器选项，添加转发IP地址操作结果如下图所示。

  2. 点击【确定】后，缓存DNS设置成功。
