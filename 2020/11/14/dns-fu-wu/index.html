0x01 DNS概述

域名如何解析

​ 根据TCP/TP中对HTTP协议的规则，在进行数据传输过程中报文需要的包含：源IP地址和目的IP地址，域名不是IP地址因此在访问时需要解析除域名对用的IP地址。

  1. 早期使用Hosts文件解析域名
    • 主机名称重复
    • 主机维护困难
  2. DNS（Domain Name System域名系统）：专门对域名进行解析的协议，安装DNS Server的服务器被称为DNS服务器。
    • 分布式
    • 层次性

域名结构

FQDN（完全合格域名）= 主机名.DNS后缀

www.jishubu.baidu.com.
www:主机名
jishubu:三级域名
baidu:二级域名
com:一级域名(顶级域名)
.:根

顶级域名:

  • 国家代码标识某个国家（如cn、uk等）
  • 国际性组织在int下
  • 国际通用
    • com：商业部门
    • net：网络服务机构
    • org：民间组织团体
    • edu：教育部门
    • gov：政府部门
    • mil：军事部门

二级域名：由各个国家自行确定

  • 类别域名（ac、com、edu、gov、net、org工6个）
  • 行政区域名（34个，如bj、js等）

三级域名：二级域名下注册的公司、单位等

主机名：服务所在服务器

DNS区域

​ ns区域是为了便于根据实际情况来分散DNS名称管理工作的负荷，将DNS名称空间划分为区域（zone）来进行管理

  • 域名空间树形结构的一部分
  • 将域名空间根据需要划分为较小区域
  • 一台DNS服务器内可存放多个区域文件
  • 一个区域所包含的范围在一个域名空间中是连续的

DNS服务器分类

  1. 主要名称服务器（主DNS）
    • 存放该区域中相关设置的DNS服务器（可修改）
    • 其存放的是区域文件的正本数据
    • 权威应答（查文件得到的结果)
  2. 辅助名称服务器（辅DNS）
    • 从主DNS服务器中复制数据
    • 数据为副本无法修改
    • 权威应答（查文件得到的结果）
  3. 主控名称服务器
    • 提供区域数据复制的DNS服务器（多个主DNS服务器谁提供辅助DNS的区域复制，谁就是主控名称服务器）
  4. Cache-Only名称服务器（缓存DNS）
    • 将查询到的数据保存在高速缓存中
    • 非权威应答（查缓存）

DNS查询方式

  1. 从查询方式分类

    • 递归查询：客户端得到结果只能是成功或失败

    • 迭代查询：

      • DNS服务器如有客户机请求数据则返回正确地址

      • DNS服务器没有请求数据则返回一个指针

  2. 从查询内容分类

    • 正向查询：由域名查找IP地址

    • 反向查询：由IP地址查找域名

DNS查询过程

  1. 输入一个HTTP网址 www.taobao.com【下单后收货时发现货物有瑕疵，计划换货】
  2. 通过DNS解析出这个网址的IP【找出卖家地址】
    1. 此时操作系统会先检查本地hosts文件是否有这个网址的映射关系，如果有，就先调用这个IP地址映射，完成解析；如果没有执行2。
    2. 若hosts里没有，则查找本地DNS解析器缓存，若有映射，直接返回，完成解析；若无映射执行3。
    3. 若前两者没有，查找TCP/IP参数中设置的首选DNS服务器（此处称为本地DNS服务器）。若查询的域名包含在本地配置区域资源中，则返回解析结果给客户机，完成解析。（此解析具有权威性）若要查询的域名，不由本地DNS服务器区域解析，但该服务器缓存了网址的映射关系，则调用这个IP地址映射，完成解析。（此解析不具权威性）。若都没有则执行4。
    4. 若前三者没有，则根据本地DNS服务器的设置（是否设置转发器）进行查询。最后都由本地DNS服务器返回IP地址给客户机。
      • 采用未转发模式：本地DNS将请求发至13台根DNS，根DNS收到后会判断这个域名.com是谁来授权管理，返回一个负责该顶级域名的服务器的IP。收到IP信息后，本地DNS联系负责.com的服务器，收到请求后进行解析，找到一个管理baidu.com的DNS服务器的IP，返回给本地DNS服务器。收到IP信息后，重复上述动作，直到找到www.taobao.com的IP地址。
      • 采用转发模式：本地DNS服务器将请求发送至上一级DNS解析器，由上一级DNS解析器进行解析。上一级若不能解析，或找根DNS或把请求转至上上级，以此循环。
  3. 有了IP后，通过IP地址找到对应服务器，请求与对方建立TCP连接（先通过子网掩码判断下是否是同一个网络的。如果不在同一个网络，需要通过网关转发）【跟客服扯皮，达成换货共识】
  4. 建立连接后，客户端向服务器发送HTTP Request（请求）包数据【寄快递】
  5. 服务器收到请求后，进行处理，返回HTTP Response（响应）包数据【卖家收到快递，寄回换的货】
  6. 客户端收到服务器的响应后，开始渲染这个Response包里的主体（body），收到全部的内容后断开与该服务器之间的TCP连接【收到货，点击确定收货，完成交易】

总结：
www.taobao.com → 本地HOSTS(无) → DNS本地缓存 → DNS服务器 → 根DNS–顶级域名DNS服务器 → 公司域名DNS服务器–找到主机对应IP → 本地DNS服务器 → 本地主机 → 完成解析得知ip

资源记录类型

资源记录 说明
SOA（起始授权机构） 定义了该域中的权威名称服务器
NS（名称服务器） 表示某区域的权威服务器和SQA中指定的该区域的主要服务器和辅助服务器
A（主机） 列出了区域中FQDN（完全合格的域名）到IP地址的映射
PTR（指针） 相对于A资源记录，PTR记录是把IP地址映射到FQDN
MX 邮件交换器记录，向指定的邮件交换主机提供消息路由
CNAME（别名） 将多个名字映射到同一台计算机上，便于用户访问

0x02 配置DNS服务

必要条件

  • 有固定的IP地址
  • 安装并启动DNS服务
  • 有区域文件，配置转发器，或者配置根提示

配置过程

  1. 安装并启动DNS服务
  2. 新建A记录
  3. 配置转发器，或者配置跟提示

0x03 管理DNS服务

子域

​ 可以将同一网络中的不同用户归于不同的子域，然后给不同资源创建不同的记录，实现不同的功能

委派

区域中的子域过多时:

  • 维护起来不方便
  • 域名查询量的瓶颈

新建委派可以将子域委派到其他服务器维护（）
子域与委派的对比

  • 子域的资源在父区域文件中，子域的权威服务器是父区域的权威服务器
  • 委派有独立的区域文件，给新域指定新的权威服务器

区域传送

  1. 区域传送（减轻单台DNS服务器负载）
    • 将一个区域文件复制到多个DNS服务器的过程
    • 从主服务器上将区域文件复制到辅助服务器

转发器

​ 具有特殊功能和应用的DNS服务器

  1. 将本地DNS服务器无法解析的查询，转发到网络上其它DNS服务器
  2. 转发到转发器的查询一般为递归查询
  3. 可以设置条件转发

根提示

  1. 使非根域的DNS服务器可查到根域DNS服务器
  2. 根提示与转发器使用顺序
    • DNS优先使用转发器

0x04 配置DNS客户端

配置静态DNS

netsh interface ip set dnsserver "本地连接" static 8.8.8.8

配置动态DNS

netsh interface ip set dnsserver "本地连接" dhcp

DNS相关命令

1.清空本地DNS缓存记录
ipconfig /flushdns

2.测试DNS服务器
nslookup

3.查询域名相关信息（https://blog.csdn.net/qq_15727809/article/details/70052534）
nslookup -qt=type 域名 DNSip或是域名
type=
A 地址记录
AAAA 地址记录
CNAME 别名记录
HINFO 硬件配置记录，包括CPU、操作系统信息
MX 邮件服务器记录
NS 名字服务器记录
PTR 反向记录
1.例如： nslookup -qt=mx baidu.com 8.8.8.8

2.nslookup -d www.baidu.com
可以看到本地ISP服务器上缓存的时间，就算刷新了本机操作系统缓存，ISP上仍然存.

3.>nslookup -qt=cname lanou.com 202.106.0.20
primary name server = ns.163.com
主要名字服务器（DNS记录文件所在的主机位置    ）

responsible mail addr = admin.NEASE.NET
记录主机管理员邮件地址admin@nease.net

serial = 20010348
区域传递序号，又叫文件版本，当发生区域复制时，该域用来指示区域信息的更新情况。
格式为：yyyymmddnn,nn代表这一天是第几次修改，辅助DNS通过比较这个号来决定是否复制该信息。大表示新。

refresh = 10800 (3 hours)
重刷新时间，当区域复制发生时，指定区域复制的更新时间间隔（告诉该区域的辅助DNS间隔多久检查该区域的数据是否是最新的）

retry = 3600 (1 hour)
重试时间，区域复制失败时，重新尝试的时间（要比刷新时间短）

expire = 360000 (4 days 4 hours)
有效时间，区域复制在有效时间内不能完成，则终止更新（设置时间要比刷新和重试时间长）

default TTL = 3600 (1 hour)
表示DNS记录在DNS服务器上缓存时间，数值越小，修改记录各地生效时间越快。
