使用WindowsServer搭建DHCP服务器

0x01 实验描述

​ 在VMware中开启三台虚拟机，其中Windows Server 2008一台命名为DHCP Server作为DHCP服务器使用，Windows 7两台分别命名为PC1和PC2作为DHCP客户端使用。DHCP Server拥有两张网卡一张为NAT模式、一张为仅主机模式，两张网卡配置如上图所示。要求PC1和PC2通过DHCP Server获取IP信息，且分配完毕后可以正常访问网络。

0x02 实验环境

  • DHCP服务器：Windows Server 2008 R2（两块网卡）
  • DHCP客户机：Windows 7（两台）

0x03 实验准备

  1. 添加名为VMnet9类型为仅主机模式的的网卡，关闭DHCP功能，并设置IP网段为：10.10.11.0。

  2. 为Windows Server添加一张新网络适配器，并设置联网模式如下图所示。

0x04 实验步骤

DHCP服务安装

  1. 打开【服务器管理器】界面，显示结果如下图所示。

  2. 点击【添加角色】选项，打开添加角色向导界面，结果如下图所示。

  3. 点击【下一步】按钮，进入服务器角色先择界面，选择DHCP服务器（若出现弹框则强制安装）。

  4. 点击【下一步】按钮后在DHCP服务器简介界面点击【下一步】按钮进入IPv4 DNS 设置界面，这里不做任何设置，直接点击【下一步】即可。

  5. 在WINS设置界面中选择【此网络上的应用程序不需要WINS】，结果如下图所示。

  6. 点击【下一步】按钮后进入DHCP作用域设置界面，此处不做设置直接点击下一步按钮即可。

  7. 点击【下一步】按钮后进入DHCPv6无状态模式界面，此处勾选禁用DHCPv6无状态模式，操作结果如下图所示。

  8. 点击【下一步】按钮，显示安装确认界面，显示结果如下图所示。

  9. 点击【安装】按钮即可开始安装DHCP服务，等待少许显示如下图所示结果表示DHCP服务已安装成功。

静态IP设置

  1. 设置NAT模式联网网卡的IP信息如下图所示。

  2. 设置仅主机联网的网卡的IP信息如下图所示，

设置NAT网卡的作用域

  1. 服务器管理器 → 角色 → DHCP服务器 → win-h69v7ls7q7d → IPv4 → 右键【新建作用域…】打开新建作用域向导界面，操作结果如下图所示。

  2. 点击【下一步】按钮后进入作用域名称设置界面，设置名称为Demo，操作结果如下图所示。

  3. 点击【下一步】按钮进入IP地址范围设置界面，操作结果如下图所示。

  4. 点击【下一步】按钮，进入添加排除和延迟界面，由于此题设置不需要设置任何选项，直接单击【下一步】按钮即可。

  5. 点击【下一步】按钮进入租用期限设置界面，设置租用期限为8天，操作结果如下图所示。

  6. 点击【下一步】按钮后进入配置DHCP选项界面，这里选择稍后配置，操作结果如下图所示。

  7. 点击【下一步】按钮进入如下所示界面，点击【完成】按钮后作用域新建完成。

  8. 作用域新建完成之后需要继续配置作用域选项，如下图可配置路由器选项。

  9. 路由器选项设置完成之后还需要设置DNS选项，操作结果如下图所示。

  10. 所有设置完成之后记得右键作用域并将其激活。

设置仅主机模式网卡作用域

  1. 服务器管理器 → 角色 → DHCP服务器 → win-h69v7ls7q7d → IPv4 → 右键【新建作用域…】打开新建作用域向导界面，操作结果如下图所示。

  2. 点击【下一步】按钮后进入作用域名称设置界面，设置名称为Only，操作结果如下图所示。

  3. 点击【下一步】按钮进入IP地址范围设置界面，操作结果如下图所示。

  4. 点击【下一步】按钮，进入添加排除和延迟界面，由于此题设置不需要设置任何选项，直接单击【下一步】按钮即可。

  5. 点击【下一步】按钮进入租用期限设置界面，设置租用期限为8天，操作结果如下图所示。

  6. 点击【下一步】按钮后进入配置DHCP选项界面，这里选择稍后配置，操作结果如下图所示。

  7. 点击【下一步】按钮进入如下所示界面，点击【完成】按钮后作用域新建完成。

  8. 在仅主机模式中虚拟机与VMnet9形成一个小局域网，且不能连接外网，所以此处不必做路由器和DNS服务器选项配置，直接右键作用域并将其激活。

0x05 实验分析

  • 此实验涉及两种网络，以NAT方式接入互联网的局域网和未接入互联网的局域网。
  • 客户端更新租期的常用名为：
    • ipconfig /release
    • ipconfig /renew
  • DHCP客户机想要连接互联网必须配置有效的网关和DNS信息。
  • 有关DHCP服务刚详细的内容可查看DHCP服务。
