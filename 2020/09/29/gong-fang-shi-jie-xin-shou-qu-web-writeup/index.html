001 view_source

0x01 题目描述

​ X老师让小宁同学查看一个网页的源代码，但小宁同学发现鼠标右键好像不管用了。

0x02 解题思路

​ 一般情况下，在浏览器的页面中右击，然后选择“检查源代码”选项，即可查看本页代码，但在页面中鼠标不能右击，则不能通过此方式查看源代码，需要通过其他方式来查看源代码。

  • 使用快捷键：F12

  • 代理抓包：Burp抓包，查看回应正文

0x03 解题工具

  • Firefox

0x04 解题步骤

  1. 访问题目网址。

  2. F12查看网页源码。

  3. 得到Flag:cyberpeace{718cca82af284e4685fb17c2416a09ff}。

0x05 考察知识

  • 浏览器源码查看方式。
  • HTML注释。

002 get_post

0x01 题目描述

​ X老师告诉小宁同学HTTP通常使用两种请求方法，你知道是哪两种吗？

0x02 解题思路

​ HTTP最常使用的两种方法分别是GET和POST方法，GET方法将参数写到URL中，POST方法将参数写入请求正文中。

  • hackbar插件

  • Burp代理（POST时无法通过）

0x03 解题工具

  • Firefox
  • HackBar

0x04 解题步骤

  1. 访问题目网址。

  2. 使用GET方式提交一个名为a，值为1的变量。

  3. 使用POST方式提交一个名为b，值为2的变量。

  4. 得到Flag：cyberpeace{338a1ac6573d4ad2bc8e31d0dd3bf8a5}

0x05 考察知识

  • GET请求
  • POST请求

  4. 得到Flag：cyberpeace{78d1de37514b1958977c5b97e393d808}

008 command_execution

0x01 题目描述

​ 小宁写了个ping功能,但没有写waf,X老师告诉她这是非常危险的，你知道为什么吗。

0x02 解题思路

​ 可能存在命令执行漏洞，尝试可执行的命令，查看flag可能存在的位置。

0x03 解题工具

  • Firefox

0x04 解题步骤

  1. 访问题目网址。

  2. 在利用命令执行漏洞是需要先知道目标系统类型，此类型使用index大小写，起送检测出目标系统为Linux/Unix系统。尝试常用命令ping 127.0.0.1 | ls。

  3. 尝试搜索flag文件。

  4. 查看flag.txt文件内容

  5. 得到Flag：cyberpeace{b93c037a4921fd7bd217f8007a6ac142}

0x05 考察知识

  • 命令执行漏洞
  • 常用操作系统命令（Windows/Linux）
