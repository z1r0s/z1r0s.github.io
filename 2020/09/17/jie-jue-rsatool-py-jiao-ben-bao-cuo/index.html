解决rsatool-py脚本报错

python rsa
debug

更新日期:   2020-10-21
文章字数:   530
阅读时长:   2 分
阅读次数:  

0x01 问题描述

​ 在解决RSA相关的Crypto问题时，需要计算推出私钥就使用rsatool.py来推到私钥。由于是第一次在本机中运行该脚本就出现了一系列与模块相关的报错信息。

0x02 运行环境

  • OS：Windows 10
  • Python2.7

0x03 解决思路

  • 首先对于模块却是问题，先使用pip管理工具进行包安装

0x04 解决步骤

  • 使用pip2 install gmpy命令来安装gmpy模块，但是仍然报错。
  • 根据提示信息使用python2.exe -m pip install --upgrade pip命令更新pip工具。
  • pip工具更新完毕后，仍使用pip2 install gmpy命令来安装gmpy模块，但依然报错。
  • 无奈之下采用本地whl文件安装，先到LDF下载相对应的whl文件。
  • 使用pip2 install C:\Users\yushe\Downloads\gmpy-1.17-cp27-none-win_amd64.whl命令完成gmpy模块的本地安装。
  • gmpy模块安装成功后再次执行python2 .\rsatool.py -o private.pem -e 65537 -p 275127860351348928173285174381581152299 -q 319576316814478949870590164193048041239命令，但又出现报错信息。
  • 使用pip2 install pyasn1命令安装pyasn1模块。
  • pyasn1模块安装后，再次执行python2 .\rsatool.py -o private.pem -e 65537 -p 275127860351348928173285174381581152299 -q 319576316814478949870590164193048041239命令，程序执行一半又出现报错信息。
  • 使用pip2 list命令查看当前已安装模块，其中pyasn1模块的版本为0.4.8。
  • 找到一篇文章发现报错原因是pyasn1库的bug，降低版本可解决此问题，执行pip2 uninstall pyasn1和pip2 install pyasn1==0.4.5命令完成pyasn10.4.5版本库的安装。
  • 再次执行python2 .\rsatool.py -o private.pem -e 65537 -p 275127860351348928173285174381581152299 -q 319576316814478949870590164193048041239命令，完美解决报错问题。

0x05 思考总结

  • 对于别人编写的脚本直接拿来用时可能需要先配置一些缺失的环境，可以使用pip直接到镜像下载安装，也可以下载whl文件在本地安装。

  • 在安装脚本的依赖环境时，最好有依赖包版本信息说明，因为版本也可能造成脚本执行失败。
